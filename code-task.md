# 今晚任务清单

## 当前情况

- 当前主开发基线为 `dev`
- `videoplayer + upload + live` 已经合并进入 `dev`
- `liveInteract` 正在由组长处理合并，暂时视为外部进行中的改动
- 前端当前核心文件主要集中在：
  - `frontend/demo/src/App.vue`
  - `frontend/demo/src/videocomponents/VideoPlayer.vue`
- 后端当前核心文件主要集中在：
  - `backend/src/main/java/com/example/demo/controller/VideoController.java`
  - `backend/src/main/java/com/example/demo/controller/CommentController.java`
  - `backend/src/main/java/com/example/demo/service/impl/VideoServiceImpl.java`
  - `backend/src/main/java/com/example/demo/service/impl/CommentServiceImpl.java`

## 今晚工作的总原则

- 不直接在不稳定的 `dev` 上做大规模功能开发
- 等 `liveInteract -> dev` 合并稳定后，再基于最新 `dev` 拉新分支
- 今晚优先完成任务梳理、问题确认、功能设计和低风险改动准备
- 所有新功能都要尽量复用现有模块，不做无意义重构
- 新建功能分支时，分支名不要使用 `codex` 前缀，统一使用正常业务语义命名
- 开发过程中必须避免乱码问题，所有新增和修改内容都要确保编码正确、中文显示正常

## 第一部分：必须优先修复的问题

### 1. 评论点赞逻辑错误

当前问题：
- 未登录用户也可以点赞评论
- 后端没有记录是谁点了哪条评论
- 同一用户可以对同一条评论重复点赞
- 点赞数会被无限累计

处理目标：
- 评论点赞必须要求登录
- 同一用户对同一条评论只能维持一个有效点赞状态
- 前后端点赞状态要保持一致

关联 issue：
- `#19 bug: 评论点赞缺少登录校验且允许重复点赞`

### 2. 评论删除缺少权限校验

当前问题：
- 任意用户都可以删除任意评论
- 后端没有校验评论作者身份
- 后端没有要求登录

处理目标：
- 删除评论必须要求登录
- 只能由评论作者删除自己的评论

### 3. 评论发布仍然会回退到默认用户

当前问题：
- 当 `userId` 缺失时，后端会默认写入用户 `1`
- 这会导致评论身份错误

处理目标：
- 评论发布必须要求登录
- 禁止用默认用户兜底

### 4. 视频点赞 / 收藏接口身份不可信

当前问题：
- 后端直接信任前端传入的 `userId`
- 未登录用户可以伪造 `userId`
- 点赞、收藏、状态查询都存在同类问题

处理目标：
- 点赞 / 收藏必须要求登录
- 后端不能直接信任前端传入的用户身份
- 视频状态查询要与真实登录用户绑定

## 第二部分：后续要做的核心功能

### 1. 个人主页模块

目标：
- 有独立的个人主页入口
- 能展示用户昵称、头像、简介
- 能展示我的上传、我的收藏、我的历史记录

需要支持的能力：
- 查询个人资料
- 编辑个人资料
- 展示个人上传内容
- 展示收藏列表
- 展示历史记录列表

建议扩展字段：
- `nickname`
- `avatar_url`
- `bio`

### 2. 关注 / 粉丝模块

目标：
- 可以关注其他用户
- 可以取消关注
- 可以查看我的关注列表
- 可以查看我的粉丝列表
- 个人主页展示关注数和粉丝数

建议新增：
- `user_follow` 表

需要接口：
- 关注用户
- 取消关注
- 查询关注列表
- 查询粉丝列表

### 3. 历史浏览记录模块

目标：
- 用户进入视频页或开始播放时自动记录浏览历史
- 用户可以在个人主页查看最近浏览记录

建议新增：
- `view_history` 表

需要接口：
- 写入浏览记录
- 查询历史记录列表

### 4. 自定义标签系统

目标：
- 上传视频时可以选择已有标签
- 上传视频时可以创建新标签
- 视频可以绑定多个标签
- 视频详情页可以展示标签

建议新增：
- `tag` 表
- `video_tag` 表

需要接口：
- 查询标签
- 创建标签
- 上传视频时绑定标签
- 查询视频标签

### 5. 通知流模块

第一版目标：
- 用户被关注时收到通知
- 用户视频收到评论时收到通知
- 用户内容被点赞时收到通知
- 用户内容被收藏时收到通知

建议新增：
- `notification` 表

需要接口：
- 查询通知列表
- 标记已读

### 6. 视频推荐模块

第一版目标：
- 首页不再只是简单返回全部视频
- 根据用户行为返回更合理的视频列表

第一版推荐依据：
- 浏览历史
- 收藏
- 点赞
- 标签
- 热门视频兜底

当前需要修改：
- `GET /api/videos/recommend` 不能继续只返回全部视频

### 7. 视频详情页整体重构

目标：
- 视频页整体布局参考 B 站常见结构
- 标题放在最上方
- 视频下方先是点赞 / 收藏 / 分享操作区
- 再往下是视频简介和标签区
- 右侧最上方是创作者卡片
- 右侧下方是相关推荐视频列表

需要支持的能力：
- 标题、播放量、发布时间等信息放在播放器上方
- 点赞 / 收藏 / 分享区独立成一行
- 视频简介支持展开 / 收起
- 标签区展示视频所有标签
- 右侧创作者卡片展示头像、昵称、简介摘要、粉丝数、关注按钮
- 点击创作者头像或昵称可以进入对应主页
- 右侧推荐区展示相关 / 推荐视频

### 8. 楼中楼评论

目标：
- 实现真正的楼中楼评论结构
- 不再使用当前这种外部 `@回复` 的伪回复方式

需要支持的能力：
- 主评论
- 子评论
- 回复某条主评论
- 回复某条子评论
- 楼内展示“回复谁”
- 支持展开更多回复
- 评论点赞要适配楼中楼结构
- 评论删除要区分主评论和子评论

建议数据设计：
- 继续使用 `parent_id`
- 必要时增加 `root_id` 或 `reply_to_user_id`，方便楼中楼查询和展示

### 9. 弹幕暂停时同步暂停

目标：
- 视频暂停时，弹幕也应立即停止运动
- 视频继续播放时，弹幕再继续同步

需要重点检查：
- 当前弹幕动画是否只是 CSS 动画独立运行
- 暂停时是否真正停止 `requestAnimationFrame`
- 暂停时已经上屏的弹幕是否还能继续移动
- 恢复播放后弹幕索引和显示状态是否正确

### 10. 上传视频时自动生成默认封面

目标：
- 用户上传视频但未手动设置封面时，系统自动生成默认封面

第一版建议：
- 优先用前端从视频首帧截取封面
- 用户可以预览自动生成的封面
- 用户可以选择接受默认封面或重新上传自定义封面

需要支持的能力：
- 上传页视频预解析
- 自动生成首帧缩略图
- 默认封面回填到上传表单
- 视频保存时把封面 URL 一并保存

### 11. 创作者主页联动

目标：
- 视频详情页中的创作者卡片可以跳转到创作者主页
- 创作者主页要成为后续个人主页能力的一部分

创作者主页建议最少展示：
- 头像
- 昵称
- 个人简介
- 粉丝数 / 关注数
- Ta 的视频列表
- 直播间入口

### 12. 分享功能第一版

目标：
- 视频页操作区包含分享能力

第一版建议：
- 先做“复制当前视频链接”
- 给出明确复制成功提示

### 13. 标签展示与推荐联动

目标：
- 标签不仅用于上传，还要进入详情页展示和推荐逻辑

需要支持的能力：
- 视频详情页展示标签
- 首页 / 右侧推荐区优先推荐同标签视频
- 标签可作为推荐、筛选、聚合的基础

### 14. 浏览历史与视频页联动

目标：
- 用户打开视频页或开始播放时写入历史记录
- 后续个人主页和推荐系统都复用这份行为数据

需要支持的能力：
- 进入视频页记录一次
- 或在首次播放时记录一次
- 历史记录展示最近浏览时间
- 后续支持跳转回原视频

## 第四部分：今晚需要先确认的重点

### 1. 先不要抢着改大前端页面

原因：
- `App.vue` 和 `VideoPlayer.vue` 都是高冲突文件
- `liveInteract` 正在合并，提前大改容易返工

### 2. 先确认后端表结构和接口设计

原因：
- 个人主页、关注、标签、历史、通知、推荐都依赖数据库结构
- 先把数据结构定清楚，后面实现效率更高

### 3. 先把现有 bug 逐条收口

原因：
- 如果基础互动逻辑本身不对，后面新增功能会建立在错误行为上

### 4. 全程注意分支命名和编码稳定性

要求：
- 不使用 `codex/*` 作为后续功能分支命名
- 分支名要直接体现功能内容，例如个人主页、推荐、历史记录等
- 修改文案、接口返回、页面文本时要特别检查中文是否正常
- 提交前要确认没有因为文件编码导致中文乱码、注释乱码、日志乱码
